SMTP / API configurations

Leave empty to keep current value: API key (Brevo, etc.), Amazon SES Secret Access Key, or saved SMTP password.

Recommended server: smtp.office365.com, port 587, encryption STARTTLS (TLS - prefilled on save). User: full Microsoft 365 email address. Password: account password or app password if MFA is enabled. Admin must enable SMTP AUTH (authenticated SMTP submission) for this mailbox or tenant. US Government (GCC High): often use smtp.office365.us as host. Keep volume reasonable: Microsoft enforces tenant-side quotas.

For accounts with EU data residency, choose "European Union" for sending and API calls. "Automatic" tries EU then US for introspection only; Symfony sending stays on US host unless EU is selected here.

AWS does not allow API calls with only the visible key (AKIA...). The Secret Access Key is required to read quotas and identities.

New template

TXT file: one address per line. Available variable: {{email}}; if a line is in First/Last/email@site.com format, you also get {{prenom}}, {{nom}}, {{name}}.

CSV file: after import, map your columns. Each recipient receives values from their own row for all variables.

  • {{prenom}} / {{first_name}} - first name
  • {{nom}} / {{last_name}} - last name
  • {{name}}, {{nom_complet}} - full name
  • {{email}} - recipient address
  • {{rotate_url}} or {{url_rotate}} - URL from a rotating list (see below)
  • {{date}}, {{time}}, {{datetime}} - send date/time
  • Random variables: {{RANDNUM-6}}, {{RANDALPHA-8}}, {{RANDALPHANUM-10}}
  • Your custom CSV columns: {{my_variable}} (lowercase and underscores)

Accepted syntax: {{variable}} or {variable}. Keys are case-insensitive.

Rotating links ({{rotate_url}})

If you use {{rotate_url}} (or {{url_rotate}}) in subject, HTML, or text, provide one URL per line. The campaign rotates these links based on the frequency below (global sent-email index).

The {{rotate_url}} variable is detected in the message - remember to fill the URL list.
